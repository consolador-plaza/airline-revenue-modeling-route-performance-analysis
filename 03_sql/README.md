# SQL Workflow

**Project:** Airline Route Performance & Estimated Passenger Revenue Analysis

This folder contains the complete SQL implementation used to transform, validate, and analyze the Bureau of Transportation Statistics (BTS) datasets for the Airline Route Performance & Estimated Passenger Revenue Analysis project.

The SQL workflow follows a structured analytics engineering pipeline, progressing from raw data preparation to business analysis. Each script has a specific responsibility and should be executed in sequence.

---

# SQL Workflow Overview

```
Raw BTS Data
        │
        ▼
01_data_preparation.sql
        │
        ▼
02_transformation.sql
        │
        ▼
03_validation.sql
        │
        ▼
04_analysis.sql
        │
        ▼
Power BI Dashboard
```

The SQL scripts collectively produce the validated analytical dataset used throughout the Power BI dashboard and supporting project documentation.

---

# Execution Order

Run the scripts in the following order.

| Order | Script | Purpose |
|------:|--------|---------|
| 1 | `01_data_preparation.sql` | Prepare and aggregate the raw BTS datasets into quarterly tables. |
| 2 | `02_transformation.sql` | Transform the quarterly datasets into the final analytical dataset (`route_performance_analytics`). |
| 3 | `03_validation.sql` | Validate data integrity, coverage, and business calculations. |
| 4 | `04_analysis.sql` | Perform exploratory and business analysis supporting the project's findings. |
| 5 | `05_route_performance_analytics.sql` | Deploy or recreate the final analytical dataset (optional deployment script). |

# Script Descriptions

## 01_data_preparation.sql

Prepare the raw BTS datasets for analytical transformation.

### Inputs

- `dbo.t100`
- `dbo.db1b_market`

### Outputs

- `dbo.t100_quarterly`
- `dbo.db1b_market_quarterly`

### Activities

- Raw data inspection
- Duplicate detection
- Quarterly aggregation
- Schema validation

---

## 02_transformation.sql

### Purpose

Transform the quarterly datasets into a unified analytical model.

### Inputs

- `dbo.t100_quarterly`
- `dbo.db1b_market_quarterly`

### Outputs

- `dbo.t100_route_quarterly`
- `dbo.db1b_market_route_quarterly`
- `dbo.route_performance_analytics`

### Activities

- Route-level aggregation

- Estimated passenger revenue calculation
- Load factor calculation
- Analytical dataset construction

## 03_validation.sql

### Purpose

Validate the analytical dataset prior to business analysis.

### Input

### Validation Categories

- Dataset completeness
- Join validation
- Route coverage
- Passenger coverage
- Weighted fare validation
- Revenue validation
- Load factor validation
- Data integrity checks

This script performs quality assurance only and does not modify analytical data.

# Business Rules

The SQL implementation follows several core business rules.

## Estimated Passenger Revenue

```
Estimated Passenger Revenue
=
Passengers × Weighted Average Fare
```

---

## Weighted Average Fare

```
Weighted Average Fare
=
SUM(Fare × Passengers)
÷
SUM(Passengers)
```

Passenger weighting prevents statistical distortion caused by simple arithmetic averages.

---

## Load Factor

```
Load Factor
=
Passengers
÷
Seats
```
